# Roadmap

The items below represent product direction, not delivery commitments.
No dates, effort estimates, contracted features, or customer commitments
are implied. Internal sprint-level planning is not included here.

## Current focus

- Closing gaps in the core registration → allocation → verification →
  check-in loop so it runs end-to-end without manual intervention
- Strengthening self-serve white-label capability, building on the
  per-tenant branding already in place
- Continuing to harden operational reporting for retailers running live
  events

## Near-term direction

- Expanding multi-store and multi-door launch support for retailers
  running a single campaign across several physical locations
- Broadening anti-abuse protections on public-facing registration
- Deepening the analytics available to retailers, both live during an
  event and after it closes

## Longer-term direction

- Public API access, so retailers can connect DropOS to their own
  systems (inventory, CRM, point of sale)
- Enterprise-readiness capabilities — including single sign-on and more
  granular permission models — for larger retail organisations
- Expansion to additional notification channels and payment integrations
  beyond what the platform supports today

## What this roadmap is not

This is a product direction, not a promise of committed dates, contracted
features, or specific customer outcomes. See the main
[`README.md`](../README.md) for the current, honest status of the
product.
